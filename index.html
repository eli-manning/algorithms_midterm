Problem 1 — Time Complexity of Matrix Multiplication

TIME ANALYSIS Three algorithms · Big-Theta · Master Theorem

SETUP

Consider the multiplication of two n × n matrices. You will analyze 3 algorithms: iterative, divide-and-conquer recursive, and Strassen's. For each, determine the time complexity.

Part A — Iterative Matrix Multiplication

THE ALGORITHM
for i = 1 to n for j = 1 to n C[i][j] = 0 for k = 1 to n C[i][j] = C[i][j] + A[i][k] * B[k][j] ← count this line
A·Q1
How many times does the innermost multiplication operation execute?
  • Outer loop (i) runs n times
  • Middle loop (j) runs n times per i → n² total (i,j) pairs
  • Inner loop (k) runs n times per (i,j) pair
  • Total = n × n × n = n³ multiplications
A·Q2
Express the running time as a function of n.

Each loop iteration performs constant work. Since the innermost line runs n³ times total, the running time is T(n) = n³.

A·Q3
Give the final time complexity using Big-Theta notation.
T(n) = Θ(n³)

Part B — Recursive Divide and Conquer

SETUP

Matrices are split into four (n/2 × n/2) blocks and multiplied recursively. This requires 8 recursive multiplications plus addition work costing Θ(n²).

B·Q1
How many recursive matrix multiplications occur?

Each n×n matrix is divided into four (n/2 × n/2) blocks. To compute the product using block multiplication, 8 recursive multiplications are performed.

B·Q2
What is the size of each recursive subproblem?

Each recursive call works on matrices of size n/2 × n/2.

B·Q3
Write the recurrence relation T(n).
T(n) = 8T(n/2) + Θ(n²)
WHY?

8 subproblems each of size n/2, plus Θ(n²) work for the matrix additions/combinations.

B·Q4
Use the Master Theorem to solve the recurrence. Show a, b, f(n), and which case applies.

Master Theorem: T(n) = aT(n/b) + f(n)

  • a = 8 (number of subproblems)
  • b = 2 (subproblem size factor)
  • f(n) = Θ(n²) (work outside recursion)

Critical exponent: logb(a) = log2(8) = 3


Compare f(n) = n² with n3:

f(n) grows slower than n³ → Case 1 applies

MASTER THEOREM CASE 1

If f(n) = O(nc−ε) for some ε > 0 where c = logb(a), then T(n) = Θ(nc)

B·Q5
State the final time complexity.
T(n) = Θ(n³)
INSIGHT

Same as the brute-force iterative algorithm! Dividing into subproblems didn't help here — still Θ(n³).

Part C — Strassen's Algorithm

SETUP

Strassen's algorithm reduces recursive multiplications from 8 → 7 using clever algebraic tricks. Matrix additions/subtractions still cost O(n²).

C·Q1
Write the recurrence relation for Strassen's algorithm.
T(n) = 7T(n/2) + Θ(n²)
DIFFERENCE FROM PART B

Only change: a = 7 instead of 8. That one fewer multiplication changes everything.

C·Q2
Identify the parameters a, b, and f(n) for the Master Theorem.
  • a = 7
  • b = 2
  • f(n) = Θ(n²)
C·Q3
Apply the Master Theorem. Compute the critical exponent and determine which case applies.

Critical exponent: log2(7) ≈ 2.807


Compare f(n) = n² with n2.807:

f(n) = n² grows slower than n2.807 → Case 1 applies

C·Q4
Determine the final time complexity.
T(n) = Θ(n^log₂7) ≈ Θ(n^2.807)
✅ KEY INSIGHT

One fewer multiplication changed the exponent from 3.0 → 2.807. For massive matrices this is a significant speedup — Strassen is asymptotically faster than standard multiplication.

📋 MASTER THEOREM CHEAT SHEET

T(n) = aT(n/b) + f(n), let c = logb(a)

  • Case 1: f(n) grows slower than nc → T(n) = Θ(nc)
  • Case 2: f(n) = Θ(nc) → T(n) = Θ(nc log n)
  • Case 3: f(n) grows faster than nc → T(n) = Θ(f(n))

State the recurrence formula. When do you include vs. exclude an item?
RECURRENCE

If Weight[i] > w (item too heavy):   P[i][w] = P[i−1][w]

Otherwise:   P[i][w] = max( P[i−1][w] , Profit[i] + P[i−1][w − Weight[i]] )

Exclude: take best profit from row above (same capacity) = P[i−1][w]

Include: add this item's profit to the best profit achievable with remaining capacity = Profit[i] + P[i−1][w − Weight[i]]

Take the max of the two options.

Part b
b
Build the Huffman tree. Show each merge step using the min-heap (priority queue).

Start: insert all as leaf nodes

Queue: b(1), a(1), n(1), l(2), o(2)

Merge 1: Take two smallest → b(1) + n(1) → internal node bn(2)

Queue: a(1), bn(2), l(2), o(2)

Merge 2: Take two smallest → a(1) + bn(2) → internal node abn(3)

Queue: l(2), o(2), abn(3)

Merge 3: Take two smallest → l(2) + o(2) → internal node lo(4)

Queue: abn(3), lo(4)

Merge 4: abn(3) + lo(4) → root(7)

FINAL TREE STRUCTURE
[7] / \ [4] [3] / \ / \ l:2 o:2 a:1 [2] / \ b:1 n:1
WHY THIS SHAPE?

l and o have the highest frequency (2), so they end up closest to the root → shortest codes. b and n are deepest → longest codes. This minimizes total bits used.

Traverse from root: left = 0, right = 1

  • root → left → right → o = 01

  • root → right → right → left → b = 110
  • root → right → right → right → n = 111
